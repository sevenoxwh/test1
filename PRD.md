# 产品需求文档 (PRD)

## 每日打卡健身小程序 (FitCheck)

**版本**: v1.0 - MVP  
**日期**: 2026-06-06  
**状态**: 待开发
**技术选型**: 微信小程序原生框架 (WXML/WXSS/JS/JSON)  
**数据存储**: 本地存储 (Storage) + 可选后端同步

---

## 1. 产品概述

### 1.1 产品定位
一款专注于"极简打卡"的微信小程序，通过降低健身记录的门槛，利用"连击天数"的心理学机制提升用户留存，并借助微信社交生态实现自然裂变。

### 1.2 核心价值主张
- **对健身新手**: 用最简单的动作（点击+选择）完成记录，告别复杂表格。
- **对自律人群**: 可视化日历与连击天数提供持续的正向反馈。
- **对社交用户**: 一键生成精美海报，满足分享成就感的需求。

### 1.3 目标用户画像 (Persona)
| 维度 | 描述 |
|------|------|
| **核心人群** | 20-35岁都市白领，有健身意识但难以坚持 |
| **痛点** | 传统App记录太繁琐，缺乏即时反馈，容易放弃 |
| **使用场景** | 健身房锻炼后、居家运动完、晨跑结束时 |
| **设备** |  primarily 手机，偶尔使用体脂秤 |

## 2. 功能需求矩阵

### 2.1 P0 - MVP核心功能 (第一迭代必须完成)

#### 2.1.1 极简打卡签到
- **用户故事**: 作为用户，我希望在锻炼结束后用最少的步骤完成打卡，并记录本次运动的类型和时长。
- **交互流程**:
  1. 用户进入首页，看到"今日已完成"大按钮（若未打卡）或"今日已打卡"状态（若已打卡）。
  2. 点击按钮后，底部弹出选择器：
     - 运动类型（单选）：有氧、力量、拉伸、游泳、瑜伽（支持全局自定义标签）
     - 运动时长（滑动选择）：15min, 30min, 45min, 60min, 90min+
  3. 可选：点击相机图标上传照片（健身房环境照、体脂秤截图、运动手表数据等）。
  4. 点击"确认打卡"，按钮状态变为已打卡，播放简短动画（如绿色对勾+撒花效果）。
- **数据存储**: 打卡记录写入本地 `wx.setStorageSync('checkins', [...])`，同时可异步同步至用户自建后端。
- **数据字段**:
  ```json
  {
    "userId": "string",
    "date": "YYYY-MM-DD",
    "sportType": "string",
    "duration": "number (分钟)",
    "photoUrl": "string (本地临时路径或后端URL)",
    "caloriesBurned": "number (自动计算)",
    "createdAt": "timestamp"
  }
  ```

#### 2.1.2 日历视图与连击激励
- **用户故事**: 作为用户，我希望在日历上直观地看到我的打卡记录，并被"连续打卡天数"激励。
- **交互设计**:
  - 首页顶部展示月度日历（7列网格）。
  - 已打卡日期显示绿色圆点徽章，当日高亮边框。
  - 日历下方显眼位置展示：
    - **当前连击天数 (Streak)**: 大号数字，如 "🔥 12 天"
    - **本月打卡次数**: "本月已打卡 8 次"
    - **最长连击记录**: "最长纪录 21 天"
- **核心逻辑**:
  - 连击计算：连续有打卡记录的自然日（允许跨月）。
  - 断连规则：超过 1 个自然日未打卡，连击归零。
  - 补卡机制 ⚠️ [待确定事项/TBD]: 是否允许补卡？如何解锁？

#### 2.1.3 个人身体档案（数据看板）
- **用户故事**: 作为用户，我希望记录我的体重变化，并看到趋势图表，直观感受坚持的效果。
- **交互设计**:
  - "我的"页面入口："身体档案"。
  - 顶部卡片：初始体重 vs 当前体重 vs 目标体重。
  - 中部图表：近 7 天 / 近 30 天体重变化折线图（使用小程序 Canvas 2D 绘制）。
  - 底部：历史记录列表，支持左滑删除。
- **数据字段**:
  ```json
  {
    "userId": "string",
    "weight": "number (kg)",
    "bodyFat": "number (%) [可选]",
    "recordDate": "YYYY-MM-DD",
    "createdAt": "timestamp"
  }
  ```

### 2.2 P1 - 快速跟进功能 (第二迭代)

#### 2.2.1 微信分享海报
- **用户故事**: 作为用户，我希望将我的打卡成就生成精美海报，分享到微信群或朋友圈。
- **交互流程**:
  1. 打卡成功后，弹出"生成分享海报"选项。
  2. 海报内容包含：
     - 用户微信头像与昵称
     - 今日运动类型与时长
     - 当前连击天数（大字突出）
     - 激励文案（随机从文案库选取）
     - 小程序码（带场景参数）
  3. 使用 Canvas 绘制海报，生成临时图片文件。
  4. 调用 `wx.showShareImageMenu` 或保存到相册后分享。
- **技术点**: Canvas 海报绘制需注意高清屏适配（DPR 处理）。

#### 2.2.2 运动类型自定义标签（全局）
- **用户故事**: 作为用户，我希望添加我特有的运动类型（如"攀岩"、"普拉提"），并让所有人都能使用。
- **交互设计**:
  - 打卡选择器底部增加"+ 自定义运动"。
  - 输入框限制 4 个汉字以内。
  - 提交后进入审核状态 ⚠️ [待确定事项/TBD]: 是否需要审核机制防止滥用？

### 2.3 P2 - 增强留存功能 (第三迭代)

#### 2.3.1 每日健身饮食建议
- **用户故事**: 作为用户，我希望在打卡后看到今日的饮食建议，帮助我更好地达到健身目标。
- **交互设计**:
  - 打卡成功页面下方展示"今日推荐食谱"卡片。
  - 卡片包含：
    - 食谱名称（如"增肌午餐：香煎鸡胸肉配藜麦"）
    - 食物图片
    - 主要食材与大致热量
    - 烹饪要点（简短文字）
  - 点击卡片进入食谱详情页（图文展示）。
- **内容来源**: 初期内置 30 套常见食谱（增肌/减脂/维持各 10 套），按用户目标体重逻辑轮播展示。

#### 2.3.2 卡路里消耗计算
- **用户故事**: 作为用户，我希望了解我今天的运动消耗了多少卡路里。
- **核心逻辑**:
  - 基于 MET（代谢当量）公式估算：
    ```
    卡路里(kcal) = MET值 × 体重(kg) × 时长(小时)
    ```
  - 内置常见运动 MET 系数表：
    | 运动类型 | MET |
    |----------|-----|
    | 有氧（慢跑） | 7.0 |
    | 力量训练 | 5.0 |
    | 拉伸/瑜伽 | 2.5 |
    | 游泳 | 8.0 |
    | 骑行 | 7.5 |
  - 体重取用户最近一次的体重记录，若无则默认 65kg。

#### 2.3.3 食物热量查询
- **用户故事**: 作为用户，我希望查询我吃的食物有多少热量。
- **交互设计**:
  - "发现"页面入口："热量查询"。
  - 搜索框输入食物名称（如"米饭"、"鸡蛋"）。
  - 展示结果列表：食物名称、每 100g 热量、常见分量参考。
- **技术方案**: 接入外部食物热量数据库 API（如薄荷健康、FatSecret 等），或使用内置常见食物表。
  ⚠️ [待确定事项/TBD]: 具体接入哪个外部数据库？是否有免费额度限制？

#### 2.3.4 AI 健身问答机器人 (FitCoach)
- **用户故事**: 作为用户，我希望随时向 AI 咨询健身相关问题，获得个性化的训练建议、饮食指导和动作纠正。
- **交互设计**:
  - **入口位置**：
    - 首页日历卡片右上角悬浮 AI 图标按钮（圆形，直径 48px，主色背景）。
    - 发现页顶部增加 "AI 教练" 功能入口卡片。
    - 个人中心功能列表增加 "AI 健身问答" 行。
  - **聊天界面**：
    - 全屏聊天页面，顶部显示 "FitCoach" 标题与状态指示器（在线/思考中）。
    - 消息气泡布局：用户消息右对齐（主色背景+白色文字），AI 消息左对齐（白色卡片+主色文字）。
    - 底部固定输入栏：多行文本输入框 + 发送按钮，支持快捷问题标签（如"减脂建议"、"增肌食谱"、"动作纠正"）。
    - 输入框上方横向滚动快捷标签栏，点击后自动发送预设问题。
  - **AI 回复内容类型**：
    - 纯文本建议（训练计划、饮食指导）。
    - 结构化卡片（如"一周训练计划"表格、"今日推荐食谱"卡片）。
    - 引用用户数据（如"根据你最近 7 天的体重趋势..."）。
  - **历史记录**：本地保存最近 50 条对话记录，支持上拉加载更多。
- **技术方案**:
  - 前端：小程序原生聊天界面，使用 `scroll-view` 实现消息列表自动滚动到底部。
  - 后端：调用大语言模型 API（如 OpenAI GPT-4 / 文心一言 / 通义千问等），通过用户自建后端服务器中转，前端不直接暴露 API Key。
  - 上下文管理：每次请求携带最近 10 轮对话历史，确保对话连贯性。
  - 用户数据注入：请求时自动注入用户最近的体重、打卡记录、目标，让 AI 回答更具个性化。
- **数据字段**:
  ```json
  {
    "role": "user | assistant",
    "content": "string",
    "contentType": "text | card | recipe",
    "timestamp": "number",
    "metadata": {
      "referencedData": "weight | checkin | goal"
    }
  }
  ```
- **合规与安全**:
  - AI 回复需添加免责声明："建议仅供参考，请根据自身情况调整，必要时咨询专业教练或医生"。
  - 禁止 AI 给出药物、医疗诊断类建议，后端需添加敏感词过滤与输出审核。

### 2.4 P3 - 社交深化功能 (第四迭代)

#### 2.4.1 订阅消息提醒
- **用户故事**: 作为用户，我希望在每天习惯的运动时间前收到提醒，避免忘记打卡。
- **交互流程**:
  1. 首次打卡后，引导用户订阅"打卡提醒"模板消息。
  2. 用户在"设置"中可自定义提醒时间（如每天 18:00）。
  3. 提醒逻辑由用户自建后端服务器的定时任务（如 node-cron / 云函数）触发，调用微信服务端 API 推送订阅消息。
- **合规点**: 必须使用 `wx.requestSubscribeMessage`，用户需主动触发订阅，不可强制。

#### 2.4.2 补卡机制
- **用户故事**: 作为用户，如果我昨天忘记打卡，我希望有机会补卡，避免连击中断。
- **交互设计**:
  - 断连后，日历页面显示"补卡"按钮（限每月 2 次）。
  - 补卡方式：观看 15 秒激励视频广告 或 分享小程序到群聊。
  - 补卡成功后，连击天数恢复。

#### 2.4.3 好友排行榜 / 打卡小队
- **用户故事**: 作为用户，我希望和好友互相监督，看到彼此的打卡情况。
- **交互设计**:
  - 微信开放数据域获取好友关系。
  - 周打卡榜：展示好友本周打卡次数排名。
  - 打卡小队：3-5 人组队，队内展示成员连击天数，全员连击达标解锁团队徽章。

## 3. 非功能性需求

### 3.1 性能要求
- 首页加载时间 < 1.5s（纯本地 Storage 读取）。
- 打卡提交响应 < 500ms（本地写入）。
- 海报生成 < 3s。

### 3.2 兼容性要求
- 基础库最低版本：2.19.4（支持云开发最新能力）。
- 适配 iOS / Android 各主流机型，重点测试刘海屏、灵动岛安全区域。

### 3.3 数据安全与隐私
- 用户体重等敏感数据存储于本地 `wx.setStorageSync`，需确保不同用户数据隔离（以 openid 为 key 前缀）。
- 照片上传至用户自建后端服务器，URL 需带鉴权参数，防止未授权访问。
- 涉及用户微信头像、昵称等开放数据，需遵循《小程序用户隐私保护指引》。

## 6. AI / 智能逻辑说明

### 6.1 卡路里计算逻辑
非 AI，纯公式计算：
```javascript
// 伪代码
function calculateCalories(sportType, durationMinutes, weightKg) {
  const metTable = {
    '有氧': 7.0,
    '力量': 5.0,
    '拉伸': 2.5,
    '游泳': 8.0,
    '骑行': 7.5,
    // ... 自定义标签需配置MET值
  };
  const met = metTable[sportType] || 5.0;
  const hours = durationMinutes / 60;
  return Math.round(met * weightKg * hours);
}
```

### 6.2 食谱推荐逻辑
非 AI，基于规则轮播：
- 根据用户目标（增肌/减脂/维持）筛选对应分类食谱。
- 每日展示 1 条，按日期取模轮播（`dayIndex % recipes.length`）。

### 6.3 海报文案库
内置激励文案数组，随机选取：
```javascript
const quotes = [
  "今天的汗水，是明天的线条。",
  "坚持不是每天都做，而是做了就不放弃。",
  "你的身体记得每一次努力。",
  // ...
];
```

### 6.4 AI 健身问答逻辑 (FitCoach)
- **系统提示词 (System Prompt)**:
  ```
  你是一位专业的健身教练和营养师，名叫 FitCoach。你的职责是为用户提供科学、安全、个性化的健身和饮食建议。
  
  规则：
  1. 回答必须基于科学健身知识，避免伪科学。
  2. 禁止提供医疗诊断、药物建议或极端节食方案。
  3. 每次回答末尾必须添加免责声明："以上建议仅供参考，请根据自身情况调整，必要时咨询专业教练或医生。"
  4. 如果用户问题超出健身/饮食范围，礼貌地引导回相关话题。
  5. 回答风格：简洁专业、鼓励性、 actionable（给出具体可执行的建议）。
  ```
- **用户数据注入**: 每次请求前，自动将用户最近 7 天打卡记录、最近体重、目标体重格式化为上下文：
  ```
  [用户数据] 最近体重: 63kg, 目标: 60kg, 最近打卡: 有氧 30min (2026-06-05), 力量 45min (2026-06-04)
  ```
- **快捷问题预设**:
  ```javascript

    { label: "减脂建议", query: "根据我的情况，给我一些减脂建议" },
  ```
